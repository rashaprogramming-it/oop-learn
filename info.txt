Класс - это описание объектов
Объект - это инсталляция (отображение) класса в памяти

---

Объектно-ориентированное программирование...

Разделяется на 3 основных принципа (на 3 парадигмы):

1) Инкапсуляция (Encapsulation)
2) Наследование (Inheritance)
3) Полиморфизм (Polymorphism)
    3.1) Параметрический полиморфизм (Parametric Polymorphism) | Истинный
    3.2) Ad-hoc полиморфизм (Ad-hoc Polymorphism) | Мнимый

---

Взаимодействия между классами: композиция, агрегация

Композиция - это когда класс A и класс B 'живут' только благодаря классу C

Агрегация - это когда класс A и класс B 'живут' только благодаря классу C, однако есть ещё один класс D, который взаимодействует с классом C, но не зависит от него, то есть если класса C не будет, то классы A и B тоже будут отсутствовать, но не класс D, - ему всё нипочём :)

---

Object-oriented programming...

It is divided into 3 main principles (into 3 paradigms):

1) Encapsulation.
2) Inheritance
3) Polymorphism
    3.1) Parametric Polymorphism True
    3.2) Ad-hoc Polymorphism | Imaginary

---

Interactions between classes: composition, aggregation

Composition is when class A and class B 'live' only because of class C

Aggregation is when class A and class B 'live' only because of class C, but there's another class D that interacts with class C but doesn't depend on it, that is, if class C doesn't exist, then classes A and B are also absent, but not class D - it doesn't care :)

---

CRUD is an acronym that stands for the four basic functions used when working with databases: create, read, update, delete.
